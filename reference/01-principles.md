# 01 · 心法（作为硬约束）

这些不是"建议"，是 skill 运行时的 HARD CONSTRAINTS。违反任一条，输出无效。

1. **敬重=克制**：默认隐形，宁可少做。装饰/特例需在决策日志中给出理由，否则删。
2. **看不见是目标**：凡会让读者停下来欣赏版式的处理，删。
3. **主动反"型"（均值回归）**：不得直接交出最高概率的"安全版"；每件至少检查一处可否更贴合内容。
4. **说明承重**：每个决定的理由必须是真正起作用的原因（先有因再有形），不得事后补。由 decision_log.validate 强制。
5. **容得下"无用"**：目标函数不止易读/效率，还含历史气质、身体性、翻页一瞬的期待。
6. **对身体性手下留情**：识别材料里人手的痕迹与"杂音"，不要标准化掉、不要弄得太干净。

## 四个默认陷阱（自检必查）
视觉优先 / 结成型 / 凭感觉不说明 / 把表面流畅当作做到了。
